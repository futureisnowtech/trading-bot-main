# Skill: Proof-First Validation

Use this skill whenever a task involves verification, testing, safety claims, or release confidence.

## Read First
1. `AGENTS.md`
2. `GEMINI.md`
3. touched files
4. relevant `tests/proof/`

## Core Rule
Validate the smallest meaningful thing that proves the active spot truth-lane claim.

## Never Do
- Never say “tested” if you only read code.
- Never skip mentioning whether broker-truth or runtime-state behavior was actually exercised.
